单例模式

2020-09-02 设计模式

# 什么是单例模式Singleton？

是指确保一个类在任何情况下都只有一个实例，提供一个全局的访问点，私有化所有构造模式，属于创建型模式。比如：ApplicationContext,ServertConfig.DBPool

# 特点:

单例模式只能有一个实例

单例类必须自己创建自己的唯一实例

提供全局的访问点

# 要素：

所有构造方法私有

指向自己实例的私有静态引用

以自己为实例返回值的静态公共方法

# 角色：

角色：单例类，访问类

# 实现方式：

  1. 饿汉模式

  2. 懒汉模式

  3. 双重检查（用于多线程）

  4. 静态内部类

  5. 枚举

# 饿汉模式

特性：安全 高效 不能懒加载

一开始就先建好，用的时候直接返回即可，如果至始至终没用使用会造成一定的资源浪费。

# 静态常量

线程安全(提前创建好，所以天生是线程安全的

static修饰，类加载的时候就会初始化，对应用的启动会造成一定程度的影响。

//静态常量
public class Singleton{
    //构造器私有化
    private Singleton(){}
    //在类内部创建对象实例
    private final static Singleton instance = new Singleton();
    //提供一个公有的静态方法，返回实例对象
    public static Singleton getInstance(){
        return instance;
    }
}

public class Singleton {
   public static final Singleton instance = new Singleton();
   private Singleton() {}
}

# 静态代码块

线程安全，类初始化时实例化

//静态代码块
public class Singleton {
   private Singleton instance = null;
   private Singleton() {}
//初始化顺序：基静态、子静态 -> 基实例代码块、基构造 -> 子实例代码块、子构造
   static {
       instance = new Singleton();
  }

   public static Singleton getInstance() {
       return this.instance;
  }
}

# 基本写法

线程不安全（并发时可能出现多个单例）

如果单例对象比较复杂，new时就比较耗时间

public class Singleton {
   private Singleton() {}//构造方法
   private static Singleton single=null;

   public static Singleton getInstance() {
        if (single == null) {  
            single = new Singleton();
        }  
       return single;
  }
}

# synchronized 同步

此时线程安全

因为加了锁，此时如果单例对象复杂，不仅耗内存，而且new的时间更长，效率更低，并且每次线程都要过来判断单例是否为空，这一也是有点耗时的。

public class Singleton {  
   private static Singleton instance;  
   private Singleton (){}

   public static synchronized Singleton getInstance() {  
  if (instance == null) {  
      instance = new Singleton();  
  }  
  return instance;  
  }  
}

双重检查锁定

线程安全,了减少锁的判断量

耗内存，而且单例对象比较复杂，比较耗时，但是对单重锁效率提升不少。

public class Singleton{
private static Singleton instance;  
   private Singleton (){}

public static Singleton getInstance() {
  //第一次检查
   if (singleton == null) {  
     synchronized (Singleton.class) {
        //第二次检查
        if (singleton == null) {  
           singleton = new Singleton();
        }  
    }  
  }  
  return singleton;
}

# 静态私有内部类（推荐）

线程安全，效率高，避免了synchronized带来的性能影响

public class Singleton {
	private Singleton() {//构造方法私有
	}
    //静态内部类里面创建了一个Singleton单例
	private static class Interr {
		private static Singleton s = new Singleton();
	}

	public static Singleton getInstance() {
		return Interr.s;
	}
}
